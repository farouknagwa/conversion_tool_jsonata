JSON QUESTION CONVERSION TOOL - EXECUTIVE SUMMARY

PURPOSE:
Automated tool to convert educational question JSON files from legacy structure to new standardized structure, with comprehensive validation and error reporting.

CAPABILITIES:
- Converts 11 question types: MCQ, MRQ, FRQ_AI, Ordering, Gap-Text, String, Opinion, Matching, Counting, Puzzle, Input-Box
- Uses JSONata transformation rules for type-specific conversions
- Three-stage pipeline: Pre-validation ‚Üí Conversion ‚Üí Post-validation
- Batch processing with progress tracking and error recovery
- Generates detailed error reports (Excel + text logs)

KEY FEATURES:
- Pre-conversion validation ensures input quality (OLD structure)
- Post-conversion validation ensures output correctness (NEW structure)
- Warning system for non-blocking issues (e.g., missing statement in multipart)
- Failed files automatically segregated by failure type
- Supports filtering by question type
- Dry-run mode for validation-only testing
- CLI interface with verbose mode
- BeautifulSoup-based HTML validation for answer fields
- Part-specific explanation extraction for multipart questions

VALIDATION SYSTEM:
- ERRORS: Block conversion, file segregated to failure folder
- WARNINGS: Logged but allow conversion to proceed
  * Tracked in separate Excel "Warnings" sheet
  * Examples: multipart without statement, single-part answer HTML issues

TYPICAL USAGE:
python3 -B main.py
(Processes all INPUT/ files with detailed progress)
